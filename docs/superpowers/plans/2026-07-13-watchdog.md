# Read-Only Watchdog Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Agent-host runs a reconcile-and-report session on a fixed interval with mutation structurally impossible (operate-loop slice 1).

**Architecture:** New `watchdog.ts` (interval scheduler with overlap guard + `WATCHDOG_PROMPT` + `watchdogDisallowedTools`); a second `SessionManager` with restricted `extraOptions`; watchdog constructed in `buildApp` and exposed via `app.locals.watchdog`; `main()` starts it. Report = a normal session titled "Watchdog — <stamp>".

**Tech Stack:** TypeScript/vitest (fake timers), agent-host only. No new dependencies.

## Global Constraints

- Spec: `docs/superpowers/specs/2026-07-13-watchdog-design.md`.
- ESM `.js` suffixes.
- The watchdog must be unable to reach the confirmation gate (gated tools disallowed by name, not gated).
- `buildApp`'s signature and existing test call sites unchanged; timers only start in `main()`.
- `RHUMB_WATCHDOG_MINUTES` unset/invalid ⇒ feature fully off.

---

### Task 1: Scheduler + disallow list + prompt (`watchdog.ts`)

**Files:** Create `agent-host/src/watchdog.ts`; Test `agent-host/test/watchdog.test.ts`.

**Interfaces:** `createWatchdog({ intervalMs: number; runTurn: () => Promise<unknown>; log?: (m: string) => void }): { start(): void; stop(): void; tick(): Promise<"ran" | "skipped"> }`; `watchdogDisallowedTools(gated: readonly string[]): string[]` (returns `["AskUserQuestion","Bash","Write","Edit","NotebookEdit", ...gated.map(t => "mcp__infra__" + t)]`); `WATCHDOG_PROMPT: string` (reconcile instructions per spec §prompt).

Steps: failing tests (vi.useFakeTimers: start → runTurn fires per interval; in-flight run ⇒ next tick returns "skipped" and runTurn called once; runTurn rejection ⇒ logged via `log`, tick resolves; stop ⇒ no further fires; disallow list includes Bash/Write/Edit/`mcp__infra__destroy_vm` and excludes `mcp__ontology__query`; WATCHDOG_PROMPT mentions report-only) → red → implement (setInterval + `.unref?.()`, `running` flag, try/catch/finally) → green → commit `feat(agent-host): watchdog scheduler with structural read-only tool list`.

### Task 2: Config

**Files:** Modify `agent-host/src/config.ts`; Test `agent-host/test/config.test.ts` (append).

**Interfaces:** `Config.watchdogMinutes: number | null` — `RHUMB_WATCHDOG_MINUTES` positive int ⇒ value, else null.

Steps: failing tests ("30" ⇒ 30; "0"/"x"/unset ⇒ null) → red → implement (`Number.parseInt`, `> 0` guard) → green → commit `feat(agent-host): RHUMB_WATCHDOG_MINUTES config`.

### Task 3: Wiring

**Files:** Modify `agent-host/src/index.ts`; Test `agent-host/test/index.smoke.test.ts` (append).

**Interfaces:** After the ontology-server merge into `sessionExtraOptions`: `const watchdogExtraOptions = { ...sessionExtraOptions, disallowedTools: watchdogDisallowedTools(GATED_TOOLS) }`; second `SessionManager` with those options; `createWatchdog` when `config.watchdogMinutes`, `runTurn` = `watchdogManager.run(WATCHDOG_PROMPT, undefined, e => { if (e.type === "session" && e.sessionId) sessions.upsertFromTurn(e.sessionId, "Watchdog — " + stamp()); })` (note: `sessions` service must be constructed before this — move its creation above if needed); attach `app.locals.watchdog`; `main()` calls `.start()` + logs cadence.

Steps: failing smoke tests (config with `watchdogMinutes: 5` ⇒ `app.locals.watchdog` defined and unconfigured ⇒ undefined; `await app.locals.watchdog.tick()` with a capturing query ⇒ captured `disallowedTools` includes `Bash` + `mcp__infra__destroy_vm`, prompt is WATCHDOG_PROMPT) → red → implement → green (full suite + build) → commit `feat(agent-host): scheduled read-only watchdog sessions`.
